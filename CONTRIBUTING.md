# Contributing Guidelines

The Labs64.IO projects accepts contributions via GitHub pull requests. This document outlines the process
to help get your contribution accepted.

## Pull Requests

Like any good open source project, we use Pull Requests (PRs) to track code changes.

### Submitting a pull request

- Fork and clone the repository
- Make your change, add tests, and make sure the tests still pass
- Create a new branch: `git checkout -b my-branch-name`
- Push to your fork and submit a pull request
- Put yourself on the back and wait for your pull request to be reviewed and merged

## Issues

Issues are used as the primary method for tracking anything to do with the Labs64.IO projects.
